H 標註降階程式

當 CMSimfly 管理者需要利用程式關鍵字代換的功能, 將各 H 標註標題頁面降階次時, 可以使用下列程式:

import re

# 打開檔案取內容
with open('config/content.htm', 'r', encoding="utf-8") as f:
    content = f.read()

rep = {"h1": "h2", "h2": "h3", "h3": "h4"} # define desired replacements here

# use these three lines to do the replacement
# re.escape(k) 在 k 字串中的特殊符號前加上反斜線
# rep.items() returns a list of rep dict's (key, value) tuple pairs
rep = dict((re.escape(k), v) for k, v in rep.items())
pattern = re.compile("|".join(rep.keys()))
#text = pattern.sub(lambda m: rep[re.escape(m.group(0))], text)
output = pattern.sub(lambda m: rep[re.escape(m.group(0))], content)
# 將 output 前置一頁 h1 後存入 content.htm
output = "<h1>期中報告</h1>" + output
with open('config/content.htm', 'w', encoding="utf-8") as f:
    f.write(output)

至於關鍵字替換升階時則需注意原頁面中已經使用 H4 作為段落標題之內容, 是否已經與先前降階之 H3 標註混合在文章內容中, 此時若選擇將所有 H4 標註升為 H3, 這些原先在內容中的 H4 標註將會被提升為分頁 H3 的標題.

對於在專案中需要進行多次升階與降階的內容, 建立在頁面標題降階之前, 保存各原先頁面標題內容與階次, 待需要升降頁面階次時, 可以用來篩選從 H4 升為 H3 流程時的參考.

期末協同專案 Q&A

Q: 是否各分組中每一位組員都必須將期末專案中的每一項任務, 從頭執行一遍?

A: 

簡短的答案為: 並非必要, 但是在期中考前的各項準備任務, 每一組員必須利用操作影片、倉儲與網站內容, 證明已經做好參與期末協同專案的準備.

此外, 本課程將依照以下的期末協同專案規劃, 在組員個人倉儲與網站、分組倉儲與網站、以及期末協同專案網站中, 透過操作影片、Github 提交歷程與內容進行自評、互評與授課教師驗證評分流程.

因為從本示範網站可以了解, 2019 Spring 協同產品設計實習課程所規劃的分組期末協同專案, 題目為「手足球系統設計與模擬」, 從實習任務二中的專案題目說明中, 所要進行的協同實習目標為:

利用 OnShape 或其他 MCAD (Mechanical Computer Aided Design) 套件, 以協同模式建立手足球系統, 並將零組件輸入 V-rep, 以建立可以兩兩競賽用的動態系統.

此處的「兩兩競賽用」包含, 兩名操作者可以透過鍵盤競賽、操作者可以與電腦程式進行競賽、以及兩組電腦程式可以進行競賽.

tinytex and pandoc.7z

Pandoc

https://pandoc.org/

使用

pandoc content.htm  --toc -t latex -V "CJKmainfont:SimSun" -V fontsize=12pt -V geometry:margin=1in --pdf-engine=xelatex -o content.pdf

可以將帶有中文字元的 content.htm 轉為 content.pdf, 但是以協同方式較不容易編輯管理 contet.htm, 因此若能先設法以 Pandoc 指令:

pandoc content.htm -t markdown -o content.md

將 content.htm 轉為 conten.md, 之後再以協同方式編輯管理此一 conten.md 檔案,  以便使用:

pandoc content.md --toc -V "CJKmainfont:SimSun" -V fontsize=12pt -V geometry:margin=1in --pdf-engine=xelatex -o content.pdf

將 content.md 轉為最後出版的 pdf 格式結案報告.

TinyTeX

https://yihui.name/tinytex/

TinyTeX 是一套縮小容量的 https://www.tug.org/texlive/, 使用者安裝後, 可以透過

tlmgr option repository http://ftp.yzu.edu.tw/CTAN/systems/texlive/tlnet 

指定安裝模組時所用的網站, 並且利用:

tlmgr search --global --file "/unicode-math.sty"

搜尋所缺少的模組, 並且利用:

tlmgr install 模組名稱

安裝所缺少的模組.
